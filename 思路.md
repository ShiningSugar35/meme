## 目前思路
  本系统分模拟交易和实盘两条链路并行，交易策略均按以下独立进行（比如模拟盘开仓仅排除当前模拟持仓）
### 买入条件
  market trenches筛选SOL链上满足以下条件的池子,每分钟轮询一次筛选新池开仓（排除当前已有持仓的）：
  x初始值：0.2
  y初始值：2.25
  
  0. 基础筛选条件
  - type new_creation 
  - platform Pump.fun / Moonshot / moonshot_app / letsbonk / memoo / token_mill / jup_studio / bags / believe / heaven 
  - min_created = 30 min
  - max_created = 240 min
  - renounced_mint = 1 
  - renounced_freeze_account = 1

  1. 风控指标
  - min_liquidity = $ (9000 - 10000 * x)
  - 0.165 - 0.1 * x ＜ top_10_holder_rate ＜ 0.26 + 0.2 * x
  - rug_ratio ＜ -0.05 + x 
  - entrapment_ratio ＜ -0.05 + x
  - is_wash_trading = 0
  - rat_trader_amount_rate ＜ -0.05 + x 
  - suspected_insider_hold_rate ＜ x
  - bundler_trader_amount_rate ＜ -0.05 + x
  - fresh_wallet_rate ＜ 0.13 + 0.1 * x
  - sell_tax ＜ 0.1 * x
  - has_at_least_one_social = 1 when x ＜ 0.15 
  - burn_status = "burn"
  - sniper_count ＜ 50 * x
  - top1(addr_type = 0)持仓份额 ＜ 0.049 + 0.01 * x

  2. 价格面及其他指标 
  - swaps_5m ＞ (4 - y) * (swaps_1h / 12) # 诞生时间不满1小时的，分母按实际分钟数/5折算并保留小数点
  - price_change_percent1h > 100 * (0.7 - 0.2 * y)
  - min-smart-degen-count = 4 - 10 * x 且满足以下条件：
    -- 持仓最多的聪明钱当前持仓份额＞1.5% 或 持仓价值＞$200， 即：amount_percentage ＞ 1.5% 或 usd_value ＞ 200 ;
    -- 持仓最少的聪明钱当前持仓份额＞1% 或 持仓价值＞$100

### 止盈止损
  调用api交易，实盘以min(1.5% liquidity,$200,钱包余额)投入，模拟盘按min(1.5% liquidity,$200)买入，以下撤仓均拿即时持仓计算比例。秒级实时监听价格：
  - 硬止盈：价格＞1.66×入盘价时，撤仓50%；＞2.2×时撤仓50%，＞2.7×时全部撤走；当 type 变为 completed 时全部撤走
  - 硬止损：价格＜0.75×入盘价时，撤仓50%；＜0.5×时全部撤走
  - 动态止损：price_change_percent1m ＜ -10% 或 price_change_percent5m ＜ -25% 时，撤仓50%
  - 时间止损：记录该池子上一次交易（初始买入或撤仓）的时间T和价格p，若10分钟内没有撤仓行为，且T+10min时刻的价格相对p满足增长率＜5%，则撤仓50%
  - 风险止损：入场按以下规则检测持仓风控指标，若不满足*1. 风控指标*的阈值，则全部撤走:当前池子持仓≥$150，每4秒扫一次*1.*；≥$100且＜$150，每8秒扫*1.*；≥$50且＜$100，每16秒扫*1.*；≥$25且＜$50，每32秒扫*1.*；＜$25，每64秒扫*1.*
  - 当持仓金额＜$12.5 时下次撤仓全部撤走
  - 当盘子里持仓的聪明钱有卖出行为时（卖出金额＞$50），撤仓50%(当前池子持仓≥$150，每4秒轮询一次；≥$100且＜$150，每8秒轮询一次；≥$50且＜$100，每16秒轮询一次；≥$25且＜$50，每32秒轮询一次；＜$25，每64秒轮询一次)

4. 模拟交易
  考虑到免费计划的api往往用量受限，先按 x=0.2,y=2.25这组进行与实盘同步的模拟交易。软件系统提供一个x、y设置列表以供模拟和实盘交易:
  - 报价阶段：调用 Jupiter API 获取最优路径和 quote。
  - 模拟阶段：将交易打包成 Jito Bundle 格式。
  - 调用 Jito simulateBundle，检测交易是否 InstructionError；若 Tip 过低，模拟失败。
  - 记录成交后的最终代币余额。
  - 动态滑点限制：据代币当前流动性池的深度和过去几分钟的真实资产波动率进行动态计算。
  - 小费：高频轮询 Jito 提供的 tip_floor REST API，提取近期成功上链的 Bundle 的实时数据流（如 25th、50th 和 75th 百分位数）。建立一套计算公式，在网络拥堵期自动匹配至 75th 百分位数，而在平静期回落至中位数水平。保持 70% 基础优先费与 30% Jito 小费的分配比例

4. 实盘交易
  Jupiter Swap API V2，优先用 Meta-Aggregator 路径。目前考虑到成本，项目先仅从两组阈值起步。同时现在先不做contextual bandit，仅仅根据预设值进行模拟和实盘交易。但保留接口，未来考虑以过去四周的实盘&模拟交易之和作为参考，倒数第四周权重10%，第三周20%，第二周30%，最近的一周40%，然后向最优阈值组合的方向步进，进行下一周的实盘交易。实盘的同时后台同步模拟其他阈值组合。（但是现在先不用开发这个）
