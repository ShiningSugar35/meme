## 目前思路

1. 风控指标 
  market trenches筛选（池子created的时间范围:[min_created,max_created]s SOL链上满足以下条件的池子,每分钟轮询一次：
  - type new_creation
  - min_liquidity = $ (5000 + 4 * min_created)
  - 0.165 - 0.1 * x ＜ top_10_holder_rate ＜ 0.26 + 0.2 * x
  - renounced_mint = 1 
  - renounced_freeze_account = 1
  - rug_ratio ＜ -0.05 + x 
  - entrapment_ratio ＜ -0.05 + x
  - is_wash_trading = 0
  - rat_trader_amount_rate ＜ -0.05 + x #项目方控盘
  - suspected_insider_hold_rate ＜ x
  - bundler_trader_amount_rate ＜ -0.05 + x
  - fresh_wallet_rate ＜ 0.13 + 0.1 * x
  - sell_tax ＜ 0.1 * x
  - has_at_least_one_social = 1 when x ＜ 0.15 
  - creator_token_status = creator_close or dev_team_hold_rate ＜ 0.03 + 0.1 * x
  - burn_status = "burn"
  - sniper_count ＜ 50 * x
  - platform Pump.fun / Moonshot / moonshot_app / letsbonk / memoo / token_mill / jup_studio / bags / believe / heaven 

2. 跟踪所有满足以上条件推送的池子，对这些池子，一分钟后再次调取上述特征，如果依旧满足上述条件。则通过GMGN调取开盘到现在，一分钟级的K-line，二筛出满足以下条件的进行最终买入(注：不满5分钟的按开盘至今计):
  - volume_1m > max(liquidity_usd*(0.07-0.02* y),median_volume_prev_5m*(1.3-0.1*y))
  - close_1m > open_1m * (1 - 0.002 * y)
  - (close_1m - low_1m) / (high_1m - low_1m) > (0.80 - 0.01 * y)
  - 当前价格 ＞ high_5m / y 
  - 当前价格 ＜ low_5m * y
  - 0.8 - 0.2 * y ＜ (当前价格 - low_5m) / (high_5m - low_5m) ＜ 0.35 + 0.2 * y
  - top1(addr_type = 0)持仓份额 ＜ 0.048 + 0.01 * x

  其中
  x初始值：0.15,0.2
  y初始值：2.25,2.50
  min_created初始值：150
  max_created初始值：240
  带"1m"或"5m"的这些字段全部取的是最近1根或5根已完成的candle

3. 止盈止损
  调用api交易，实盘以min(1.5% liquidity,$200,钱包余额)投入，模拟盘按min(1.5% liquidity,$200)买入，撤仓拿即时持仓计算，实时监听价格：
  - 硬止盈：价格＞1.8 × 入盘价时，撤仓50%；＞2.5×时撤仓50%，＞3.1×时全部撤走；当 type 变为 completed 时全部撤走
  - 硬止损：价格＜0.8 × 入盘价时，撤仓50%；＜0.6×时全部撤走
  - 动态止损：价格 ＜ low_1m 时，撤仓50%
  - 时间止损：记录该池子上一次交易（初始买入或撤仓）的时间T和价格p，若5分钟内没有撤仓行为，且T+5min时刻的价格相对p满足增长率＜15%，则撤仓50%
  - 风险止损：入场按以下规则检测持仓风控指标，若不满足**1.风控指标**的阈值，则全部撤走:当前池子持仓≥ $150，每两秒扫一次**1.**，同时每10秒扫一次top1(addr_type = 0)持仓份额（须满足＜ 0.048 + 0.01 * x）；≥$100且＜$150，每三秒扫**1.**，同时每15秒扫一次top1(addr_type = 0)；≥$50且＜$100，每六秒扫**1.**，同时每30秒扫一次top1；≥$25且＜$50，每12秒扫**1.**，同时每60秒扫一次top1；＜$25，每24秒扫**1.**
  - 当持仓金额＜$12.5 时下次撤仓全部撤走

4. 模拟交易
  考虑到免费计划的api往往用量受限，先按 x=0.2,y=2.25,min_created = 150,max_created = 240 这组进行与实盘同步的模拟交易。软件系统提供一个x、y、min_created、max_created设置列表以供模拟和实盘交易:
  - 报价阶段：调用 Jupiter API 获取最优路径和 quote。
  - 模拟阶段：将交易打包成 Jito Bundle 格式。
  - 调用 Jito simulateBundle，检测交易是否 InstructionError；若 Tip 过低，模拟失败。
  - 记录成交后的最终代币余额。
  - 动态滑点限制：据代币当前流动性池的深度和过去几分钟的真实资产波动率进行动态计算。
  - 小费：高频轮询 Jito 提供的 tip_floor REST API，提取近期成功上链的 Bundle 的实时数据流（如 25th、50th 和 75th 百分位数）。建立一套计算公式，在网络拥堵期自动匹配至 75th 百分位数，而在平静期回落至中位数水平。保持 70% 基础优先费与 30% Jito 小费的分配比例

5. 实盘交易
  Jupiter Swap API V2，优先用 Meta-Aggregator 路径。目前考虑到成本，项目先仅从两组阈值起步。同时现在先不做contextual bandit，仅仅根据预设值进行模拟和实盘交易。但保留接口，未来考虑以过去四周的实盘&模拟交易之和作为参考，倒数第四周权重10%，第三周20%，第二周30%，最近的一周40%，然后向最优阈值组合的方向步进，进行下一周的实盘交易。实盘的同时后台同步模拟其他阈值组合。（但是现在先不用开发这个）
