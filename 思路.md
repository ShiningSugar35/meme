## 目前思路

1. 风控指标 
  market trenches筛选SOL链上满足以下筛选条件的池子,每分钟轮询一次：
  - type new_creation
  - min_liquidity = 13000 - 20000 * x
  - 0.175 - 0.15 * x ＜ top_10_holder_rate ＜ 0.25 + 0.25 * x
  - 0.0335 - 0.02 * x ＜ top1_holder_rate ＜ 0.044 + 0.04 * x
  - renounced_mint = 1 
  - renounced_freeze_account = 1
  - max_rug_ratio ＜ -0.05 + x 
  - max_insider_ratio ＜ -0.05 + x
  - max_entrapment_ratio ＜ -0.05 + x
  - is_wash_trading = 0
  - rat_trader_amount_rate ＜ -0.05 + x #项目方控盘
  - suspected_insider_hold_rate ＜ x
  - max_bundler_rate ＜ -0.05 + x
  - fresh_wallet_rate ＜ 0.13 + 0.1 * x
  - sell_tax ＜ 0.1 * x
  - has_social = 1 when x ≤ 0.3 
  - creator_token_status = creator_close or dev_team_hold_rate ＜ 0.04 + 0.1 * x
  - dev_token_burn_ratio ＞ 1 - 0.1 * x  
  - sniper_count ＜ 50 * x
  - min-created t s
  - max-created (t+60) s
  - platform Pump.fun / Moonshot / moonshot_app / letsbonk / memoo / token_mill / jup_studio / bags / believe / heaven 

2. 跟踪所有满足以上条件推送的池子，对这些池子，一分钟后再次调取上述特征，如果依旧满足上述条件。则通过GMGN调取开盘到现在，一分钟级的K-line，二筛出满足以下条件的进行最终买入:
  - 过去一分钟: buy_volume_1m > sell_volume_1m * (1.2 - y * 0.05)
  - 当前价格 ＞ 过去5分钟的价格最高点 / y （不满5分钟的按开盘至今计）
  - 当前价格 ＜ 过去5分钟的价格最低点 * y
  - 0.8 - 0.2 * y ＜ (当前价格 - 过去5分钟最低点) / (过去5分钟最高点 - 过去5分钟最低点) ＜ 0.35 + 0.2 * y

  x初始值：0.2，0.15
  y初始值：2.25，2.75
  t初始值：150,510

3. 止盈止损
  调用api模拟交易，本金以min(1.25% liquidity,2 SOL)投入，撤仓拿即时持仓计算，实时监听价格：
  - 硬止盈：价格＞1.8×入盘价时，撤仓50%；＞2.5×时撤仓50%，＞3.1×时全部撤走；当 type 变为 completed 时全部撤走
  - 硬止损：价格＜0.8×入盘价时，撤仓50%；＜0.6×时全部撤走
  - 动态止盈：价格＞2×过去一分钟最低价时，撤仓50%；＞3×时，全部撤走
  - 动态止损：价格＜0.75×过去一分钟最高价时，撤仓50%；＜0.55×时，全部撤走
  - 时间止损：记录该池子上一次交易（初始买入或撤仓）的时间T和价格p，若5分钟内没有撤仓行为，且T+5min时刻的价格相对p满足增长率＜15%，则全部撤走
  - 风险止损：入场按以下规则检测持仓风控指标，若不满足**1.**的阈值，则全部撤走:当前池子持仓≥1.5 SOL，每两秒扫一次；≥1.0 SOL，每四秒；≥0.5 SOL，每八秒；≥0.25 SOL，每16秒；＜0.25 SOL，每32秒
  - 当持仓金额＜0.125 SOL时下次撤仓全部撤走

4. 模拟交易
  考虑到免费计划的api往往用量受限，先按 x=0.15,y=2.25,t=150 和 x=0.2,y=2.75,t=510 这两组进行与实盘同步的模拟交易。软件系统提供一个x、y、t模拟值设置列表（包含三组）以供模拟交易:
  - 报价阶段：调用 Jupiter API 获取最优路径和 quote。
  - 模拟阶段：将交易打包成 Jito Bundle 格式。
  - 调用 Jito simulateBundle，检测交易是否 InstructionError；若 Tip 过低，模拟失败。
  - 记录成交后的最终代币余额。
  - 动态滑点限制：据代币当前流动性池的深度和过去几分钟的真实资产波动率进行动态计算。
  - 小费：高频轮询 Jito 提供的 tip_floor REST API，提取近期成功上链的 Bundle 的实时数据流（如 25th、50th 和 75th 百分位数）。建立一套计算公式，在网络拥堵期自动匹配至 75th 百分位数，而在平静期回落至中位数水平。保持 70% 基础优先费与 30% Jito 小费的分配比例

5. 实盘交易
  Jupiter Swap API V2，优先用 Meta-Aggregator 路径。目前考虑到成本，项目先仅从两组阈值起步。同时现在先不做contextual bandit，仅仅根据预设值进行模拟和实盘交易。但保留接口，未来考虑以过去四周的实盘&模拟交易之和作为参考，倒数第四周权重10%，第三周20%，第二周30%，最近的一周40%，然后向最优阈值组合的方向步进，进行下一周的实盘交易。实盘的同时后台同步模拟其他阈值组合。（但是现在先不用开发这个）
