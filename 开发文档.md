## 运行模式（用户视角）

用户只看到两个模式，通过前端一键切换，不需要改 .env：

| 模式 | 说明 | 前端标识 |
|------|------|----------|
| 模拟盘测试 (SIM_TEST) | 只跑模拟，不真实广播，默认模式 | Sim Test Only |
| 模拟+实盘正式交易 (FORMAL_SIM_LIVE) | 模拟+实盘同时运行，需满足全部安全条件 | Sim + Live Trading |

底层 ProviderMode (mock/online_readonly/live) 对用户不可见，由 runtime settings 控制。

## 数据库变更

### 新增表
- `runtime_settings`: 存储运行模式、worker 开关、实盘开关等前端可控配置

### 修改字段
- `positions`: 新增 `account_type` (LIVE/SIM)、`legacy_config_status`、`updated_at`、`next_check_at`、`last_checked_at`、`pnl_pct`
- `trade_events`: 新增 `account_type`
- `system_events`: 新增 `account_type`
- `token_metric_snapshots`: 新增 `source_mode` (MOCK/REAL)

### 新增索引
- positions: account_type, token_mint, next_check_at, updated_at
- trade_events: account_type, token_mint
- system_events: level, category, account_type

### 迁移说明
- 存量 `is_live=1` 的 position/trade_event 自动回填 `account_type='LIVE'`
- 存量 invalid locked_strategy_config_json 自动标记 `legacy_config_status='LEGACY_INVALID_CONFIG'`，不再每轮重复刷屏
- 前端 "Ops & Emergency" 页面提供一键修复入口

## 前端页面重构

旧有 7 个页面合并为 3 个：

| 页面 | 路由 | 功能 |
|------|------|------|
| Control Center | `/` | 模式切换、Provider 健康、Worker 状态、Kill Switch、策略配置、摘要卡片 |
| Portfolio | `/portfolio` | LIVE 持仓表、SIM 持仓表（上下分离），支持排序/展开详情 |
| Ops & Emergency | `/ops` | 系统日志（可选 level/module 过滤）、Trade Ledger、Provider Requests、紧急操作按钮（二次确认） |

### 表格列定义
- Status: OPEN/CLOSED/PENDING_ENTRY/PENDING_EXIT/FAILED/BLOCKED，含短 mint
- Entry $: 入场价格 (USD)
- Remaining: 剩余 USD 估值
- Price: 最新 price_usd
- Liquidity: GMGN/Jupiter 流动性
- PnL %: 后端统一计算
- MCap: market_cap

## 后台 Worker 体系

### Worker Manager
`backend/app/services/worker_manager.py`: 统一管理后台异步 runner 循环

| Worker | 间隔 | 职责 |
|--------|------|------|
| discovery | 60s | GMGN trenches 发现 + 初筛 |
| second_filter | 30s | 二筛 + Jupiter 验证 |
| price_monitor | 5s | PriceAggregator 价格采集 |
| position_risk | 10s | 动态扫描频率，dust 强平，风控重检 |
| kill_switch | 30s | Rolling 10 ROI 止损 |

每个 worker 独立 try/except，异常不杀死 FastAPI，记录 system_event。

### Worker 状态
- running, last_run_at, last_error, processed_count
- 支持 start/stop/status，禁止重复启动
- 优雅 shutdown (CancelledError)

## 实盘交易安全门

### 进入实盘条件（全部满足）
1. .env 中 `LIVE_TRADING_ENABLED=true`
2. `DRY_RUN=false`
3. `JITO_ENABLED=true`
4. `WALLET_PUBLIC_KEY` 已配置
5. `WALLET_PRIVATE_KEY_BASE58` 已配置
6. GMGN/Jupiter API key 已配置
7. RPC URL 已配置
8. Kill switch 未触发
9. 前端确认切换到 FORMAL_SIM_LIVE

### 交易执行前 final safety gate
- PROVIDER_MODE=mock 直接放行（模拟测试）
- 非 mock 时：DRY_RUN=true → blocked
- LIVE_TRADING_ENABLED=false → blocked
- JITO_ENABLED=false → blocked
- WALLET_PUBLIC_KEY 缺失 → blocked
- Jito 不可用时严禁 fallback RPC 广播

### 严防事项
- 私钥不在日志/前端/测试/providers/system_events 中出现
- 不使用 MOCK_PUBKEY 作为 fallback
- 不支持 `SOL` 字符串作为 Jupiter inputMint，必须使用 WSOL mint (`So11111111111111111111111111111111111111112`)
- 实盘交易有幂等键防止重试重复买入

## Provider 改动

### GMGN Provider
- 新增 `source_mode` 字段标记 MOCK/REAL
- Mock token (PASS1/FAIL_INIT/FAIL_SECOND) 不在非 mock 模式查询真实 API

### Jupiter Provider
- quote_exact_in 始终使用 WSOL mint
- build_swap_instructions: LIVE 模式调用真实 API

### Jito Provider
- online_readonly/mock 模式 BLOCK send()
- LIVE 模式实现 tip ladder 重试 (50th→75th→95th)
- 同步 send() 不可用时无 RPC fallback

### Trading Pipeline
- wallet_pubkey 必检：缺失则直接返回错误，不使用 MOCK_PUBKEY
- 所有 trade_event 带 account_type (LIVE/SIM)
- live_entries_enabled 在 MOCK 模式下总是放行
- idempotency key 防止重复买入

## 性能优化

- DB 新增 10+ 个索引覆盖 account_type, status, token_mint, next_check_at
- PositionRiskRunner: 按 remaining_value_sol 分 5 档动态调整扫描频率
- PositionRiskRunner: legacy_config_status 标记后不再每轮重复检查
- SecondFilterRunner: 非 mock 模式跳过 MOCK_MINTS 集合
- Provider health: 不再每次请求初始化新 DB/provider
- 前端轮询频率按页面可见性调整 (Control Center 3s, Portfolio 5s, Ops 4s)
- Worker 启动/停止由前端控制，不自动运行
