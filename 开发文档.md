# 开发文档

## 运行模式

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DRY_RUN` | `true` | 阻止真实交易广播 |
| `LIVE_TRADING_ENABLED` | `false` | 是否允许实盘 |
| `PROVIDER_MODE` | `mock` | mock / online_readonly / live |

安全门逻辑：`mock` 模式放行（模拟交易测试）；非 `mock` 模式下，`DRY_RUN=true` 或 `LIVE_TRADING_ENABLED=false` 或 `JITO_ENABLED=false` 均阻止广播。

## API Key 动态扫描

调用 `load_dotenv()` 后直接从 `os.environ` 扫描，无硬编码数量上限：

- `GMGN_API_KEY_\d+` + `GMGN_PUBLIC_KEY_\d+` + `GMGN_PRIVATE_KEY_\d+`
- `JUPITER_API_KEY_\d+`
- `ANKR_API_KEY_\d+`

新增或删除 key 只需修改 `.env`，无需改代码。向后兼容方法 `get_gmgn_api_key()` 返回第一组可用 key。

## PriceAggregator

三层降级链：

1. **GMGN subscription** (WebSocket) — 最快，最准
2. **GMGN latest price** (HTTP) — 中等
3. **Jupiter quote** (fallback) — 最后手段，标记 `JUPITER_QUOTE_FALLBACK`

每次 tick 都写入 `tick_snapshots`，`source` 字段记录实际来源。

## Runner 体系

### DiscoveryRunner

- 每 60s 轮询 GMGN trenches
- snapshot_id 去重（同 snapshot+token+pool 不重复创建 discovery event）
- 通过 EventBus 发布系统事件

### SecondFilterRunner

- 对已发现 token 执行二筛
- 检查 pool_created_at 时间窗口
- 通过策略组配置（x, y, t）进行多策略评分

### PriceMonitorRunner

- 对 open position 使用 PriceAggregator 采集实时价格
- 写入 tick_snapshots

### PositionRiskRunner

- 动态扫描频率：根据 `remaining_value_sol` 分为 5 档（2s / 4s / 8s / 16s / 32s）
- Dust 强平：`remaining_value_sol < DUST_FORCE_EXIT_SOL (0.125)` 时 100% 退出
- 风控重检：locked_strategy_config_json 重新跑 filters，不通过则平仓
- 滚动窗口 120s 用于 exit_rules 决策

### KillSwitchRunner

- 计算最近 10 笔已平仓 live position 的 rolling ROI
- `rolling_10_roi <= -0.20` 时设置 `pause_new_entries = True`
- 可通过 `/api/risk/kill-switch/reset` 重置

## EventBus + SSE

`services/event_bus.py`：异步 pub/sub，per-channel 队列（max 100），asyncio.Lock 线程安全。

`/api/logs/stream`：Starlette StreamingResponse，每 30s ping 保活。

## 交易执行管线

### BUY 流程

```
安全门检查 → kill switch → 同 token open live position 检查
→ GMGN latest price → compute_entry_size
→ Jupiter quote ExactIn → priceImpactPct <= cap 检查
→ build swap instructions → Jito simulate → Jito send (tip ladder)
→ 写 pending trade_event → RPC wait_signature
→ 写 confirmed trade_event → create_position → bandit_observation
```

幂等键：`BUY:{token_mint}:{strategy_id}:{config_version}:{snapshot_id}`

### SELL 流程

```
安全门检查 → 读 position → 按 remaining_token_amount 计算卖出量
→ Jupiter quote → priceImpact 检查
→ 写 pending trade_event → Jito send → RPC confirmation
→ close_position (全平) 或 update_position_remaining (部分平)
```

幂等键：`SELL:{position_id}:{exit_reason}:{timestamp_bucket}`

### Jito Tip Ladder

50th → 75th → 95th 百分位，最多 retry 2 次。InstructionError 不重试，tip too low 才抬 tip。

## 安全注意事项

1. `.env` 在 `.gitignore` 中，永不提交
2. 所有 provider request log 脱敏：不记录完整 API key、private key、raw tx、auth header
3. trade_event 的 `quote_json` 只存摘要（impact 等字段），不存完整 quote
4. `DRY_RUN=true` 默认拦截所有真实广播
5. RPC `sendTransaction` / `sendRawTransaction` 在 mock/online_readonly 下直接抛异常
6. Jito 不可用时无 RPC fallback
