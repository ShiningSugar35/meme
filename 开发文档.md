## 项目总目标

本项目是一个可真实接入 GMGN + Jupiter + Jito + Solana RPC 的工程原型。必须实现实盘交易路径，但必须通过 `.env` 和前端开关控制是否启用真实广播，避免误触发。模拟交易只是同步记录与评估，不得替代实盘功能。

系统核心目标：

基于 GMGN 发现 Solana 新池，按策略组参数 `x/y/t` 做初筛与二筛。二筛通过后立即调用 Jupiter 获取交易路径并构造交易，通过 Jito 发送实盘交易；同时后台对其他策略组做同步模拟。持仓后以秒级价格监听驱动止盈止损、风险清仓、小额残仓清理、过去 10 单亏损强停。所有策略、行情、决策、交易、异常都必须落 SQLite，并保留未来 contextual bandit 调用所需的 feature vector 与 reward 数据。

## 二、严禁事项

1. 严禁把任何 API key、GMGN private key、wallet private key 写入代码、README、日志、前端 bundle 或测试 fixture。
2. 严禁打印完整 API key、私钥、签名交易原文、base64 transaction、seed phrase。
3. 严禁实现所谓“全局 x”。本项目没有全局 x。
4. 严禁实现所谓 `entry_x / entry_y / entry_t` 字段。正确做法是仓位绑定 `locked_strategy_config_json`，即买入时对应策略组完整配置快照。
5. 严禁在二筛通过后再做“quote 前轻量二筛”。真实流程是：初筛 → 等待 60 秒 → 二筛 → 立即 quote/build/buy。
6. 严禁同一个 token 被多个 live 策略组重复买入。只能实盘买一次。
7. 严禁把退出百分比按初始仓位计算。所有撤仓比例永远按当前剩余仓位计算。
8. 严禁 Jito 不可用时自动 fallback 普通 RPC 实盘广播。v1 默认不 fallback。
9. 严禁 GMGN 失败时使用旧缓存继续买入。GMGN 本轮失败则跳过本轮发现/筛选。
10. 严禁把 contextual bandit 真正上线自动调参。v1 只保留数据结构和接口，不做自动调参。

## 四、系统架构

系统由四层组成：

```text
Frontend React/Vite
  ↓ REST / SSE
FastAPI Backend
  ↓ service layer
Asyncio Task Runners
  ↓ providers
GMGN / Jupiter / Jito / Solana RPC / Optional DexScreener
  ↓
SQLite WAL
```

核心异步 runner：

1. `DiscoveryRunner`

   * 每 60 秒调用 GMGN Trenches / market 数据。
   * 只使用 GMGN。
   * 仅处理 SOL 链新池。
   * 只保存第一次进入候选池的 `first_seen_at`。
   * 重复命中只追加 snapshot，不重复创建候选。

2. `SecondFilterRunner`

   * 对已进入候选池的 token，在当前时间距离 pool 创建时间落入 `[t, t+60]` 时执行二筛。
   * 二筛使用 GMGN latest price 和从开盘到当前/过去 5 分钟的 1m K-line。
   * 至少需要 2 个价格点。
   * `high == low` 直接失败。
   * 二筛通过后立即进入 quote/build/buy pipeline。
   * 不做 quote 前轻量二筛。

3. `PriceMonitorRunner`

   * 对 active position 每 1 秒拉取 GMGN 实时价格。
   * 写入 `tick_snapshots`。
   * 所有策略组共享同一份 tick。
   * 如果 GMGN 秒级价格不可用，才用 Jupiter Price 或 DexScreener 做备用展示/校验，不作为主风控源。
   * RPC WebSocket 用于签名状态、余额、账户变化，不作为直接价格 oracle。

4. `PositionRiskRunner`

   * 对实盘和模拟仓位执行统一退出规则。
   * 多条退出规则同时触发时，取 `desired_exit_pct` 的最大值。
   * 撤仓比例永远按当前剩余仓位计算。
   * 失败进入 retry，不阻塞更高优先级清仓。
   * 剩余价值 `< $10` 时下一次撤仓直接 100%。

5. `KillSwitchRunner`

   * 只针对 live closed positions。
   * 最近 10 笔 live 仓位：
     `rolling_10_roi = total_return_sol / total_cost_sol - 1`
   * 若 `rolling_10_roi <= -0.20`，暂停新开仓。
   * 强停后继续监控已有仓位并继续风控清仓。

Token 候选状态：

```text

SECOND_FILTER_FAILED
QUOTE_PENDING

POSITION_OPEN

EXIT_PENDING
SELL_RETRYING
CLOSED

```

重要状态规则：

* 二筛失败允许后续重新进入，不永久拉黑。
* GMGN 请求失败：跳过本轮，不使用旧缓存买入。
* Jupiter / RPC / Jito 请求失败：不追单，但按 retry policy 记录失败和重试。
* `InstructionError`：直接失败，不继续抬 tip 重试。
* `tip too low`：按 50th → 75th → 95th 重试，最多 2 次。
* 同一个 token 多个 live 策略组同时命中：按 `strategy_groups.priority` 最小值执行唯一一次实盘买入；其他策略组只记录模拟 decision / simulated position。

## 七、Provider 抽象接口

实现 `providers/base.py`：

```python
from abc import ABC, abstractmethod
from typing import Any, Dict, List, Optional

class MarketDataProvider(ABC):
    @abstractmethod
    async def fetch_trenches(self, params: Dict[str, Any]) -> List[Dict[str, Any]]:
        ...

    @abstractmethod
    async def fetch_token_snapshot(self, token_mint: str) -> Dict[str, Any]:
        ...

    @abstractmethod
    async def fetch_kline(self, token_mint: str, interval: str, limit: int) -> List[Dict[str, Any]]:
        ...

    @abstractmethod
    async def fetch_latest_price(self, token_mint: str) -> Dict[str, Any]:
        ...

class SwapProvider(ABC):
    @abstractmethod
    async def quote_exact_in(self, input_mint: str, output_mint: str, amount: int, slippage_bps: int) -> Dict[str, Any]:
        ...

    @abstractmethod
    async def build_swap_instructions(self, quote: Dict[str, Any], user_public_key: str, extra: Dict[str, Any]) -> Dict[str, Any]:
        ...

class ExecutionProvider(ABC):
    @abstractmethod
    async def simulate(self, transaction_or_bundle: Any) -> Dict[str, Any]:
        ...

    @abstractmethod
    async def send(self, transaction_or_bundle: Any) -> Dict[str, Any]:
        ...

class RpcProvider(ABC):
    @abstractmethod
    async def get_balance(self, wallet: str) -> Dict[str, Any]:
        ...

    @abstractmethod
    async def get_token_balance(self, wallet: str, mint: str) -> Dict[str, Any]:
        ...

    @abstractmethod
    async def wait_signature(self, signature: str, timeout_seconds: int) -> Dict[str, Any]:
        ...
```

具体实现：

* `gmgn.py`：主数据源，负责 trenches、token snapshot、K-line、latest price。
* `jupiter.py`：负责 quote 和 build raw instructions。
* `jito.py`：负责 tip_floor 缓存、simulate、send。
* `rpc.py`：负责余额、token account、signatureSubscribe / getSignatureStatuses。
* `dexscreener.py`：可选 backup，不参与主交易判断。

## 九、二筛公式

实现 `strategy/second_filter.py`。

输入：

* token
* strategy_group.y
* GMGN latest price
* 开盘至今或最近 5 分钟 1m K-line
* 过去 1 分钟 buy/sell volume

硬约束：

* 至少 2 个价格点。
* `high_5m == low_5m` 时直接失败，避免分母为 0。
* 当前价格使用 GMGN latest price。
* quote price 只用于交易执行，不用于二筛。
* 二筛通过后立即进入 Jupiter quote/build/buy，不做任何“轻量二筛”。

规则：

```text
buy_volume_1m > sell_volume_1m * (1.2 - y * 0.05)

current_price > high_5m / y

current_price < low_5m * y

0.8 - 0.2 * y
<
(current_price - low_5m) / (high_5m - low_5m)
<
0.35 + 0.2 * y
```

注意：这里 `y` 的默认值可能大于 1，例如 2.25、2.5、2.75。不要擅自把 y 当成 0-1 系数。

## 十、买入 sizing

实现 `strategy/sizing.py`。

买入固定 SOL，ExactIn。

```python
entry_size_sol = min(0.0125 * sol_side_liquidity, 2.0)
```

注意：

* 使用 SOL 侧流动性，不是总 USD liquidity。
* 如果只有 USD liquidity 而没有 SOL side liquidity，要在 `sol_side_liquidity_missing` 里标记，并默认不实盘买入；可以记录模拟失败。
* Jupiter amount 使用 lamports。
* 保留 `entry_size_usd` 展示字段，但风控 sizing 以 SOL 为主。

## 十二、交易执行路径

### 实盘买入


## 十四、持仓退出规则

实现 `strategy/exit_rules.py`。

输入：

* position
* current tick
* rolling 60s tick window
* latest GMGN snapshot
* locked_strategy_config_json
* current remaining amount/value

输出：

```python
ExitDecision(
    should_exit: bool,
    exit_pct: float,
    reasons: list[ExitReason],
    emergency: bool
)
```

核心规则：

```text
硬止盈：
price > 1.8 * entry_price → 50%
price > 2.5 * entry_price → 50%
price > 3.1 * entry_price → 100%

硬止损：
price < 0.8 * entry_price → 50%
price < 0.6 * entry_price → 100%

动态止盈：
price > 2 * rolling_60s_low_tick → 50%
price > 3 * rolling_60s_low_tick → 100%

动态止损：
price < 0.75 * rolling_60s_high_tick → 50%
price < 0.55 * rolling_60s_high_tick → 100%

时间止损：
T = 上一次实际成交确认时间
p = 上一次实际成交确认价格
如果 now >= T + 5min 且期间没有撤仓行为，
并且 current_price / p - 1 < 0.15 → 100%

风险止损：
入场后每 60s 用 locked_strategy_config_json 中的 x 重新检查初筛风控指标。
若不满足入场风控阈值 → 100%

completed 清仓：
type == completed → 100%

小额残仓：
remaining_value_usd < 10 → 下一次撤仓 100%
```

多规则同时触发：

```python
exit_pct = max(rule.desired_exit_pct for triggered rule)
```

绝对硬约束：

```text
所有百分比永远按当前剩余仓位计算。
不要用初始仓位计算。
```

例如当前只剩 40% 仓位，触发 50% 退出，则卖出当前剩余仓位的一半，即初始仓位的 20%。

## 十五、过去 10 单亏损强停
